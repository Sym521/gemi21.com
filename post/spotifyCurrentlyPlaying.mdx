自身の備忘録も兼ねて，Spotifyの「現在再生中の曲」を表示するAPIの実装方法をまとめます．
Next.jsをOpenNext経由でCloudflareにデプロイしている環境で，Spotify APIのrefresh tokenを使って再生中の曲名とアーティスト名を返す構成です．

# 実装の概要
今回作ったAPIは，Spotifyで曲を再生しているときに曲名とアーティスト名を返し，再生していないときは空の状態を返すものです．
このサイトのトップページとFooterで使用しています．

```spotify.ts
const clientId = process.env.SPOTIFY_CLIENT_ID;
const clientSecret = process.env.SPOTIFY_CLIENT_SECRET;
const refresh_token = process.env.SPOTIFY_REFRESH_TOKEN;

async function getAccessToken() {
    if (!refresh_token) {
        throw new Error("SPOTIFY_REFRESH_TOKEN is not defined");
    }

    const tokenResponse = await fetch("https://accounts.spotify.com/api/token", {
        method: "POST",
        headers: {
            Authorization: `Basic ${Buffer.from(
                `${clientId}:${clientSecret}`
            ).toString("base64")}`,
            "Content-Type": "application/x-www-form-urlencoded",
        },
        body: new URLSearchParams({
            grant_type: "refresh_token",
            refresh_token: refresh_token!,
        }),
    });
    return tokenResponse.json();
};


export async function getCurrentlyPlayingTrack() {
    try {
        const tokenData = await getAccessToken();
        const accessToken = tokenData.access_token;

        if (!accessToken) {
            throw new Error("Failed to get access token");
        }

        const response = await fetch("https://api.spotify.com/v1/me/player/currently-playing", {
            headers: {
                "Authorization": `Bearer ${accessToken}`,
            },
        });

        if (response.status === 204) {
            // 何も再生していない場合
            return null;
        }

        if (!response.ok) {
            throw new Error(`Spotify API error: ${response.status}`);
        }

        return await response.json();
    } catch (error) {
        console.error("Error fetching currently playing track:", error);
        return null;
    }
}
```

レスポンスは以下のような形にしています．

```json
{
  "isPlaying": true,
  "name": "Song name",
  "artist": "Artist name"
}
```

再生中でない場合は以下のように返します．

```json
{
  "isPlaying": false,
  "name": "",
  "artist": ""
}
```

# Refresh Tokenの取得
Spotify APIを継続的に使うためには，最初にauthorization codeを取得し，それを使ってrefresh tokenを取得します．
詳細は<a href='https://developer.spotify.com/documentation/web-api/tutorials/refreshing-tokens' className='text-blue-600 hover:underline'>公式ドキュメント</a>を確認してください．

## Authorization Codeの取得
WindowsならPowerShellで以下を実行して，Spotifyの認可URLを作成します．

```powershell
$CLIENT_ID = "SpotifyのClient ID"
$REDIRECT_URI = "http://127.0.0.1:3000/callback"
$SCOPE = "user-read-currently-playing user-read-playback-state"
$STATE = [guid]::NewGuid().ToString("N")

$authUrl = "https://accounts.spotify.com/authorize" +
  "?client_id=$([uri]::EscapeDataString($CLIENT_ID))" +
  "&response_type=code" +
  "&redirect_uri=$([uri]::EscapeDataString($REDIRECT_URI))" +
  "&scope=$([uri]::EscapeDataString($SCOPE))" +
  "&state=$STATE" +
  "&show_dialog=true"

Start-Process $authUrl
```

ブラウザで許可すると，最終的に以下のようなURLに遷移します．

<CodeBlock
    language="txt"
    code={`http://127.0.0.1:3000/callback?code=xxxxx&state=yyyyy`}
/>

ここで`127.0.0.1 で接続が拒否されました`と表示されても問題ありません．ローカルサーバーを立てていないため正常です．
コピーするのは<span className='text-rose-600'>code=の後ろだけ</span>です．`&state=...`は含めません．

## Token交換
取得したcodeを使ってrefresh tokenを取得します．
codeは一度しか使えないので，失敗した場合は新しいcodeを取り直します．

```powershell
$CLIENT_ID = "SpotifyのClient ID"
$CLIENT_SECRET = "SpotifyのClient Secret"
$REDIRECT_URI = "http://127.0.0.1:3000/callback"
$CODE = "取得したcodeだけ"

$basic = [Convert]::ToBase64String(
  [Text.Encoding]::UTF8.GetBytes("${CLIENT_ID}:${CLIENT_SECRET}")
)

$body = "grant_type=authorization_code" +
  "&code=$([uri]::EscapeDataString($CODE.Trim()))" +
  "&redirect_uri=$([uri]::EscapeDataString($REDIRECT_URI))"

$response = Invoke-RestMethod `
  -Method Post `
  -Uri "https://accounts.spotify.com/api/token" `
  -Headers @{ Authorization = "Basic $basic" } `
  -ContentType "application/x-www-form-urlencoded" `
  -Body $body

$response.refresh_token
```

表示された値を`SPOTIFY_REFRESH_TOKEN`としてCloudflare Secretに登録します．

# よくあるエラー

## `SPOTIFY_REFRESH_TOKEN is not defined`
CloudflareのRuntime Secretに`SPOTIFY_REFRESH_TOKEN`が登録されていない，または本番環境に反映されていない

ローカルの`.env`だけに値を入れていても，Cloudflare上では読めません．対象のWorkerにSecretが入っているか確認しましょう．

## `invalid_client`
`SPOTIFY_CLIENT_ID`または`SPOTIFY_CLIENT_SECRET`が間違っています．
別のSpotifyアプリのClient IDとClient Secretを混ぜている場合にも起きます．

## `invalid_grant: Invalid authorization code`
authorization codeが無効です．

主な原因は以下です．

- codeを一度使った
- 古いcodeを使った
- `code=`や`&state=...`まで含めて貼った
- 認可URLとtoken交換時の`redirect_uri`が一致していない

対処としては，新しいcodeを取り直して，すぐに一度だけ使います．

## `invalid_grant: Invalid refresh token`
Cloudflareに登録されている`SPOTIFY_REFRESH_TOKEN`が無効です．

主な原因は以下です．

- 別のSpotifyアプリ由来のrefresh tokenを使っている
- Client Secretを再生成した
- Spotifyアカウント側でアプリ連携を解除した
- 古いrefresh tokenを登録している

この場合は，新しいauthorization codeからrefresh tokenを取得し直し，Cloudflare Secretを上書きします．
